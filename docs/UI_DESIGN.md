# 后台管理系统 UI/UX 设计规划

## 1. 文档定位

本文档定义后台管理系统的最终页面结构、设计体验规则、模块对应功能和后续开发边界。

本文档最初用于完成网站设计和用户体验底座：

- 做完整后台壳子、菜单、路由、页面结构和交互状态。
- 保留已接入的登录、用户管理、权限管理、审计日志。
- 未接接口的页面必须明确显示“待接入接口”，不能伪装成已完成。

当前项目已经在设计底座之上接入了大量真实业务接口。后续维护本文档时，应同时标注页面是“真实接入”“第一版状态流转”“占位待接入”还是“后期增强”，不能只按早期设计阶段理解。

## 2. 设计原则

### 2.1 后台风格

- 采用专业后台管理系统布局：左侧深色菜单、顶部栏、内容工作区。
- 页面优先支持高频操作、快速筛选、批量处理和风险提醒。
- UI 不做营销页，不做装饰性大图，不做与业务无关的视觉噱头。
- 表格、抽屉、弹窗、筛选、状态标签是核心体验组件。

### 2.2 业务隔离

- Apple ID 代充业务和兑换码自动发货业务必须在菜单、页面、表格字段、报表入口上清晰分区。
- 可共用客户、来源平台、权限、日志、附件和系统通知。
- 发货模板归兑换码业务，只服务兑换码发货和售后补发；通知模板归系统配置，不作为业务发货话术。
- 不共用业务订单、成本逻辑、库存逻辑、报表逻辑。

### 2.3 模块化开发

- 侧边栏菜单由模块配置驱动。
- 每个模块定义标题、路由、业务区、阶段、状态、权限点、页面能力和表格字段。
- 后续开发业务时，优先替换当前页面的数据源和接口，不重新设计页面骨架。
- 新模块必须先补菜单配置、权限点、测试用例和文档，再写业务代码。

## 3. 设计稿处理结论

### 3.1 保留

- 深色侧边栏 + 顶部栏 + 工作区的整体后台结构。
- 首页仪表盘、续费工作台、Apple ID 操作计划、Apple ID 管理、兑换码库存、兑换码订单、通知设置、安全中心、数据中心、运维监控、网站维护等页面方向。
- 表格、筛选、状态标签、右侧详情抽屉、弹窗、指标卡、加载和空状态。

### 3.2 合并

- “权限管理”与现有角色权限功能合并。
- “审计日志中心”与现有操作日志功能合并，后续再补敏感查看、导出、权限变更、自动化和平台接口日志。
- “发货模板”归兑换码自动发货，用来给兑换码客户发送交付话术；“通知模板”归系统配置，用来做后台提醒。
- “平台状态”和“平台接口日志”分开：前者看连接健康，后者看调用记录。

### 3.3 不进入生产菜单

- 设计规范页不放到生产侧边栏。
- 设计规范沉淀在本文档和可复用 UI 组件中。

### 3.4 后期增强

- 风控中心、客服工单、上线检查清单保留为后期增强模块。
- 这些模块不阻塞 Apple ID 核心业务和兑换码发货核心业务。

## 4. 全局布局

### 4.1 侧边栏

侧边栏按业务区和工作场景细分，保持“一级主分类 + 页面入口”的两级结构，一级分类可展开/收起：

- 工作台
- 客户与来源
- ID 业务
- 兑换码业务
- 安全与风控
- 数据与审计
- 运维与平台
- 系统配置

侧边栏必须支持：

- 一级分类展开/收起，展开后直接显示页面入口，不再增加中间小组层级
- 一级分类采用手风琴交互，同一时间只展开一个主分类
- 一级分类左侧展示清晰可识别的语义图标，图标本身必须可见，不能只放弱边框占位
- 当前页面所在分类自动展开
- 展开状态本地持久化
- 菜单搜索
- 搜索时自动展开匹配分类并隐藏无关页面
- 当前页面高亮
- 状态徽标
- 一级分类右侧数字只表示当前分类下有几个页面入口，不表示待处理数量。
- 待处理、异常、未读、失败等消息提醒显示在主导航模块右上角；用户处理完成后，提醒数必须跟随真实接口刷新减少。
- 同一个主导航如果既有页面数量又有提醒数量，页面数量保持右侧小数字，提醒数量使用更醒目的角标，避免两种数字混在一起。
- 移动端抽屉式展开

### 4.2 顶部栏

顶部栏包含：

- 当前页面所属分组
- 当前页面标题
- 全局搜索入口
- 通知入口
- 刷新入口
- 当前用户菜单

全局搜索第一版只做入口预留，后续可接入客户、订单、ID、兑换码、任务搜索。

### 4.3 内容区

所有页面统一使用：

- 顶部栏唯一页面标题
- 标题右侧说明按钮
- 右侧工作区页签，用于切换已打开页面
- 主要操作按钮
- 指标卡片
- 表格/表单/详情抽屉
- 正文顶部操作区只放真实业务操作，不展示“已接入接口”“Phase”等开发验收状态

工作区页签规则：

- 点击左侧导航后，右侧顶部自动增加对应页面页签
- 已打开页面可以在右侧顶部直接切换
- 页签切换不得重置当前页面内未提交的表单、筛选和临时输入状态
- 页签数量超过可视宽度时横向滚动，不挤压顶部栏和正文内容
- 页签条右侧提供“关闭当前 / 关闭其他 / 关闭全部”操作
- 关闭全部清空顶部页签条并释放旧页面缓存，不清空当前页面内容；再次点击左侧导航后重新打开页签
- 复杂报表页的多维度表格只挂载当前可见维度，避免隐藏表格造成页面卡顿

## 5. 通用组件

### 5.1 页面骨架

`PageScaffold` 用于所有后台页面，统一正文操作区和内容布局。页面标题只在顶部栏展示，页面说明通过顶部标题右侧的问号按钮查看，避免正文区域重复标题和副标题。正文顶部操作区只承载真实业务按钮，开发阶段、接口接入状态和页面分组信息不在业务页面顶部展示。

### 5.2 指标卡

`MetricCard` 用于首页、报表、监控、任务中心等页面。

指标卡必须包含：

- 指标名称
- 数值
- 辅助说明
- 状态色

### 5.3 表格工具栏

所有列表页默认具备：

- 搜索
- 筛选
- 日期快捷筛选：今天、昨天、近7天、近30天、本月、上月、自定义
- 刷新
- 列显示配置
- 密度切换
- 主操作按钮
- 筛选标签回显
- 分页

### 5.4 状态组件

所有模块必须具备：

- 加载骨架屏
- 空状态
- 错误状态
- 无权限状态
- 保存中状态
- 成功提示
- 失败提示
- 危险操作二次确认

### 5.5 右侧抽屉

右侧抽屉用于高频详情和处理动作：

- Apple ID 详情
- 续费任务处理
- Apple ID 操作计划
- 兑换码详情
- 发货异常处理
- 售后补发
- 通知规则
- 敏感字段审批
- 平台接口配置

## 7. 关键交互规则

### 7.1 列表页

- 搜索、筛选、排序、分页、刷新、导出、列配置、密度切换、保存视图入口必须统一。
- 筛选后要显示筛选标签回显。
- 表格在移动端允许横向滚动，不能挤压到文字重叠。
- 敏感字段默认脱敏，完整查看必须经过权限和审计。

### 7.2 表单页

- 必填项必须有校验。
- 提交时按钮显示保存中状态。
- 关闭未保存表单时应提示确认。
- 保存成功显示 toast。
- 保存失败显示错误提示，并保留用户输入。

### 7.3 危险操作

以下操作必须二次确认：

- 删除或永久清理数据
- 强制下线会话
- 修改角色权限
- 查看完整敏感字段
- 批量导出数据
- 释放兑换码锁定
- 重跑自动化任务
- 开启维护模式

### 7.4 敏感信息

敏感字段包括：

- Apple ID 密码、密保、备用邮箱、恢复信息
- Apple ID 充值记录里的礼品卡代码完整内容
- 兑换码完整内容
- Telegram Bot Token
- 平台 Token 和授权信息

显示规则：

- 列表只显示脱敏值或尾号。
- 查看完整内容需要权限、原因和审计日志。
- 高风险查看可进入审批流程。

## 8. 后续开发落地顺序

1. 保持当前设计壳稳定，不再大改全局布局。
2. 先围绕 `docs/ROADMAP_TO_LAUNCH.md` 修复上线阻塞项。
3. 真实接入页面继续复用现有组件，不重复设计页面骨架。
4. 第一版状态流转页面要明确边界，例如平台授权占位、Apple ID 自动化 Worker 未接入、数据中心物理执行器未接入。
5. 每接入一个模块，同步更新权限、API 文档、测试用例、操作日志和上线路线图。
6. 每个模块完成后进行移动端和表格溢出检查。

## 9. 验收清单

- 登录页可用。
- 侧边栏菜单完整。
- 每个菜单都有页面。
- 已开发页面仍能访问真实接口。
- 未开发页面显示专业占位和待接入状态。
- 表格、抽屉、弹窗、toast、加载、空状态、错误状态可以预览。
- 移动端侧边栏可收起，表格不重叠。
- 设计规范不进入生产菜单。
- Apple ID 和兑换码业务边界清晰。

## 10. V3 设计还原验收

视觉还原、响应式和交互体验的逐页验收，以 `docs/V3_DESIGN_QA.md` 为准。

后续每一轮页面修复都必须按 V3 验收清单检查：

- 桌面 1440x900。
- 移动端 390x844。
- 页面首屏是否存在无意义空白。
- 卡片、按钮、表格、弹窗、抽屉是否符合 V3 tokens。
- 是否仍有明显 Element Plus 默认视觉。
- 是否仍有开发占位文案直接暴露在正式业务主视觉中。
