# 第一版上线策略

## 1. 结论

第一版采用 **半自动可运营优先** 策略。

配置口径：

```env
FIRST_RELEASE_MODE=semi_auto
```

这表示系统先保证后台完整闭环、人工可控、日志可审计、失败可转人工；淘宝/闲鱼真实开放平台和 Apple ID 真实自动化 Worker 不作为第一版上线阻断项。

## 2. 选择原因

- 当前系统已经具备 Apple ID 代充和兑换码业务的核心后台闭环。
- 淘宝/闲鱼真实 OAuth、签名、订单同步、发货、退款同步依赖平台开放能力和真实账号配置，不能在没有平台授权资料时伪造完成。
- Apple ID 自动化涉及真实账号登录、风控、人机验证和设备环境，第一版必须保留人工验证兜底。
- 半自动策略能先上线内部使用，沉淀真实订单、真实库存、真实成本和操作日志，再逐步替换平台 Adapter 和自动化 Worker。

## 3. 第一版必须满足

- Apple ID 业务数据、订单、成本、报表与兑换码业务隔离。
- 兑换码订单支持手工录入、库存锁定、半自动发货、售后补发和利润报表。
- 淘宝/闲鱼页面保留同步、发货、退款入口，但真实开放平台未接入时必须明确转人工或记录失败，不允许伪装成功。
- 平台轮询任务可以开启，但真实 Adapter 未接入前只用于验证日志、通知和运维链路。
- 高风险操作必须写审计日志。
- 敏感字段必须加密、脱敏、按权限查看，并写敏感查看日志。
- 数据备份、恢复演练、MFA、安全中心、运维监控和上线检查清单必须可用。

## 4. 不阻断第一版上线

- 淘宝真实 OAuth、签名、订单同步、发货、退款同步。
- 闲鱼真实授权、订单同步、电子凭证或无需物流发货。
- 平台 Webhook 的真实验签和回调字段映射。
- Apple ID 真实自动化 Worker。
- 蓝绿发布或灰度发布。

这些能力进入上线后增强，但对应的接口、页面、日志和失败转人工底座已经保留。

## 5. 切换到全自动上线的条件

如果后续决定改为 **必须全自动发货后再上线**，需要把：

```env
FIRST_RELEASE_MODE=full_auto
```

并完成：

- 淘宝真实 OAuth、签名、订单同步、发货、退款同步。
- 闲鱼真实授权、订单同步、电子凭证或无需物流发货。
- 平台 Webhook 或轮询任务接入真实 Adapter。
- Apple ID 真实自动化 Worker。
- 平台授权过期、接口失败、发货失败、自动化验证失败的告警和重试验证。

在 `full_auto` 模式下，Phase 17 的 T1702、T1703、T1705、T1706 必须完成后才能进入正式上线。
